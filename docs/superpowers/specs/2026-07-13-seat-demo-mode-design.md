# 五席位演示模式设计

## 目标

新增独立的“席位演示模式”。演示者可从右上角在五个席位账号间切换；各席位展示与职责相符的默认页面。只有指挥决策席与情报处理席可使用檀棋，并在同一演习环节中按固定顺序生成各自负责的结构化报告。

## 范围与隔离

- 保留现有原型模式（`standard`）和全流程演示模式（`full-flow`）的页面、数据与交互。
- 新增 `seat-demo` 模式及单独的持久化 store；不复用或写入 `full-flow-demo` 的运行数据。
- 进入席位演示模式时重置该模式自己的报告、任务和航线显示状态；退出时不清理其他两种模式的数据。
- 现有演示模式入口继续负责原型模式与全流程模式的切换；新增独立的席位演示入口，避免三种模式循环切换造成误操作。

## 账号与默认页面

账号显示名直接使用席位名，账号切换只在席位演示模式出现。切换账号后关闭檀棋浮窗与右侧檀棋面板，跳转到该席位默认页面。

| 账号（席位） | 默认页面 | 能力边界 | 檀棋 |
| --- | --- | --- | --- |
| 指挥决策席 | `/action` | 行动列表、设备卡片、状态与态势地图；可下发作战方案，不提供方案调整 | 作战方案报告（RW）、毁伤评估报告（HS）、作战效能评估报告（PG） |
| 任务规划席 | `/wayline` | 航线推演和航线下发；不提供航线调整 | 不显示 |
| 情报处理席 | `/situation/events` | 目标上图、车辆检测结果、态势地图与设备图像数据；不提供目标融合或识别结果调整 | 态势报告（TS）及后续新增的非 RW/HS/PG 报告 |
| 综合管理席 | `/sources?type=UAV` | 资源、设备数据、兵力部署、回放与地图标绘；不提供链路监控、汇总调整或存储设置 | 不显示 |
| 综合显示席 | `/source/UAV` | 设备资源、状态、载荷数据、设备卡片、驾驶舱和地图；设备与驾驶舱通过既有页面手动切换 | 不显示 |

## 共享环节与报告顺序

### 当前环节

- 席位演示模式以两条预置行动作为环节一和环节二的数据源；行动详情页被打开时，将该行动设为所有账号共享的当前环节。
- 账号切换不改变当前环节。所有账号对同一环节读取同一份全局报告游标、任务列表和已公开航线。
- 环节一、环节二均完成后，下一次由指挥决策席提交的消息生成环节三的作战效能评估报告。

### 会话和游标

- 每个环节只有一个全局报告游标，保证整个演示不跳步、不重复。
- 每个檀棋席位拥有独立会话视图：指挥决策席只看 RW/HS/PG，情报处理席只看 TS 等情报类报告。
- 当前待生成报告不属于已登录席位时，输入区禁用并提示应切换到哪个席位；不生成额外的模型文字，避免破坏“模型只输出报告”的规则。
- 当前待生成报告属于已登录席位时，任意非空输入都会记录为该席位的提问，并生成下一份预置报告。

### 报告归属

| 既有报告类型 | 席位 | 席位模式展示名称 |
| --- | --- | --- |
| `task` | 指挥决策席 | 作战方案报告 |
| `damage` | 指挥决策席 | 毁伤评估报告 |
| `evaluation` | 指挥决策席 | 作战效能评估报告 |
| `situation` | 情报处理席 | 态势报告 |
| `inventory` | 情报处理席 | 装备清单报告 |

## 任务和航线

- 指挥决策席的作战方案报告继续使用既有“执行任务”确认弹窗。
- 点击“立即执行”后，仅在席位演示模式自己的状态中创建对应任务、将任务置为执行中，并公开相应航线。
- 任务规划席可在航线页面看到同一环节已公开的航线；其他模式的任务和航线数据不受影响。
- 已执行的同一报告再次确认不创建重复任务或重复航线。

## 技术结构

- `seat-demo.logic.ts`：无 UI 依赖的纯函数，负责报告归属、下一报告拥有者、游标推进和环节完成判断；以 Node 原生测试覆盖。
- `seat-demo.store.ts`：席位模式的账号、当前环节、会话、报告游标、任务与航线状态，使用本地持久化并具备版本迁移。
- `SeatTanqiDemo.tsx`：席位专用檀棋面板。保持当前檀棋视觉样式，处理账号权限、输入禁用和报告展示名称。
- `SeatDemoAccountMenu.tsx`：在既有右上角用户菜单内增加五席位切换，只在 `seat-demo` 模式渲染。
- 现有行动、任务、航线服务模块仅增加 `seat-demo` 分支，继续保留 `full-flow` 与标准路径。

## 验收标准

1. 三个模式可以反复切换，彼此的演示状态互不影响。
2. 仅席位演示模式显示五个席位账号；每次切换跳转至指定默认页面。
3. 只有指挥决策席、情报处理席可以打开檀棋。
4. 同一环节跨账号共享报告顺序，无法由错误席位越过下一份报告。
5. 每个檀棋席位仅看到其负责的报告，会话切换后仍保留。
6. 作战方案确认下发后任务与航线只出现一次，并可被任务规划席查看。
7. `pnpm ts`、`pnpm lint` 与 GitHub Pages 构建通过；手工浏览器检查账号切换、报告顺序和任务航线联动。
